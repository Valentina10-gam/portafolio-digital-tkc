# TKC — Instrucciones permanentes del proyecto

## Objetivo

Crear desde cero un micrositio interactivo y un portafolio comercial digital para TKC Fumigaciones Group, diseñado para navegación normal y presentación en un evento de networking.

## Fuente única de contenido

El archivo:

content/TKC_CONTENIDO_APROBADO_NO_MODIFICAR.md

contiene el contenido comercial aprobado.

Está prohibido:

- Modificar ese archivo.
- Corregirlo.
- Resumirlo.
- Reescribirlo.
- Sustituir frases.
- Eliminar contenido.
- Inventar títulos.
- Inventar servicios.
- Inventar cifras.
- Inventar sectores.
- Inventar procesos.
- Crear textos comerciales alternativos dentro de los componentes.

La presentación y diagramación pueden cambiar, pero el contenido debe conservarse.

## Recursos oficiales

- Logos institucionales: public/brand/
- Logos de clientes: public/clients/
- Única fotografía permitida: public/images/contacto-tkc.jpg
- Manual, documentos e inspiración: references/

No modificar los logos oficiales.

No:

- Convertirlos a blanco.
- Cambiar sus colores.
- Recortarlos.
- Deformarlos.
- Recrearlos con texto, CSS o SVG.
- Aplicar filtros de inversión.

## Fotografía

Solo puede utilizarse una fotografía en todo el sitio:

public/images/contacto-tkc.jpg

Debe aparecer únicamente en la sección final de contacto.

## Referencias visuales

Las imágenes de references/inspiracion/ sirven únicamente para estudiar:

- Composición.
- Ritmo.
- Profundidad.
- Espacio.
- Motion.
- Transiciones.
- Jerarquía.

No deben insertarse ni copiarse literalmente dentro del sitio.

## Dirección visual

- Experiencia principalmente clara.
- No crear una web completamente morada.
- No crear dashboards.
- No repetir tarjetas rectangulares en todas las secciones.
- No usar el patrón triangular como wallpaper general.
- Cada línea de negocio debe tener un ambiente propio.
- El logo de cada línea funciona como título de su sección.
- No repetir el nombre con un heading gigante adicional.
- Debe sentirse moderna, editorial, dinámica y premium.
- Debe funcionar en 1920 × 1080 para un evento.

## Desarrollo por fases

No construir todo el sitio en una sola iteración.

Fase 1:

1. Auditoría de recursos.
2. Arquitectura técnica.
3. Sistema de diseño.
4. Header.
5. Hero.
6. Quiénes somos.
7. Ecosistema interactivo.
8. Event Mode funcional.

Detenerse al finalizar la Fase 1 y esperar aprobación.

## Validación

Antes de finalizar cada fase:

- Ejecutar build.
- Revisar errores de consola.
- Comprobar responsive.
- Comprobar navegación por teclado.
- Comprobar reduced motion.
- Informar cambios y pendientes reales.
